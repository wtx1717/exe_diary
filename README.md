# exe_diary

一个面向跑步训练的桌面端笔记记录软件。项目目标是每天定时获取用户在 Garmin Connect 上的运动数据，保存到本地，并通过弹窗向用户收集简短主观反馈，形成可长期追踪的训练日志。

## 项目目标

- 自动同步 Garmin 跑步运动数据，减少手动记录成本。
- 在运动后或每天固定时间弹出简短问题，记录身体状态、疲劳、睡眠、训练感受等主观信息。
- 将客观运动数据和主观反馈合并存储，方便后续查询、统计和复盘。
- 优先保证本地可用、数据可控，再逐步扩展可视化、训练建议和多设备同步。

## 核心流程

1. 程序在电脑端后台运行或由系统计划任务定时启动。
2. 程序登录或访问 Garmin Connect，获取最新运动记录。
3. 程序将运动数据写入本地数据库或文件。
4. 程序弹出简洁窗口，询问用户当天状态和训练感受。
5. 程序把用户回答与对应运动记录关联，形成一条训练日志。

## 第一版候选功能

- Garmin 运动数据抓取：跑步类型、时间、距离、配速、心率、步频、训练效果等。
- 本地数据存储：先支持本地数据库，便于后续查询和统计。
- 定时运行：每天固定时间检查新增运动。
- 运动后确认：发现新增运动后弹窗提示用户补充主观记录。
- 简短问卷：疲劳程度、疼痛情况、睡眠质量、训练感受、备注。
- 基础日志视图：查看历史运动和对应笔记。

## 待确认问题

- Garmin 数据来源：使用官方 API、第三方库，还是通过网页自动化/爬取。
- 登录方式：是否需要保存账号凭据，是否支持手动登录后复用会话。
- 运行方式：常驻后台程序，还是 Windows 计划任务定时启动。
- 数据存储：SQLite、本地 JSON/CSV，或后续支持云端同步。
- 弹窗时机：运动完成后尽快弹出，还是每天固定时间集中询问。
- 第一版界面：只做弹窗和日志列表，还是同时做统计面板。

## 开发原则

- 优先做稳定、可验证的最小版本。
- 敏感信息尽量本地保存，并避免明文存储账号密码。
- 数据结构预留扩展空间，便于后续加入训练计划、图表和分析。
- 先把数据采集和记录闭环跑通，再扩展复杂建议功能。
